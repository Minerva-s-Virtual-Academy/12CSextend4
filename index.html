1. Code Rules Cheat Sheet

Before starting the task, review these basic structural patterns:

  • Constants: Variables that do not change. Write them in capital letters: MY_VALUE = 10.
  • Data Conversion: Change data types using functions like int("5").
  • Error Trapping: Stop your program from crashing when it gets bad data using a try-except block: 
    try:
    number = int(item)
except ValueError:
    # Code to run if item is not a number
  • Bitwise AND (&): Compares binary bits. For example, 7 & 15 results in 7.
2. Practical Workspace: Network Data Checker

Instruction: Read the task description below. Complete the code inside the black editor box, then click the check button.

The Task:
You need to write a program that filters a list of mixed data sent over a network.

Your code must:
  1. Create a constant named BIT_MASK and set it to 15.
  2. Complete the function named validate_packet(payload_list).
  3. Inside the function, look at each item in payload_list using a loop.
  4. Try to convert each item into a whole number. If it fails because the data is text, add 1 to an error_count tracking variable and move to the next item.
  5. If it is a number, check if it is smaller than 0 or larger than 255. If it is out of bounds, skip it.
  6. Apply your BIT_MASK to the valid number using the & operator. If the result is exactly 0, add the number to a list named safe_list.
  7. The function must return three things: a True/False status flag (True if there were no errors at all, False if there were), the safe_list, and the final error_count.

A network router receives this list of data items: [32, "Error", 12, 500]. The programmer wants to clean this data before processing it.
